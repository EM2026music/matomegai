# GPTへの指示書（まとめ買いリスト アプリの見た目リニューアル）

このメッセージと一緒に `index.html` を渡します。**見た目（デザイン・UI）だけを新しくしてください。**
機能・データ・動作は今のまま1ミリも変えないでください。

---

## このアプリは何？

スマホで使う「1週間のまとめ買い」支援アプリです。単一のHTMLファイル1枚（`index.html`）だけで動きます。
下タブで4つの画面を切り替えます。

1. **📅献立表** … 月〜日の各曜日に料理を割り当てる。上部に「家族の人数」の−／＋ボタン。
2. **🍳献立** … 料理テンプレ（約190品）の一覧。タップで材料・分量・「何人分か」を編集。新規作成・コピー（食べ盛り版）・削除・バックアップ保存/読み込みもここ。
3. **🧻日用品** … トイレットペーパー等のボタンをタップで選択（緑）。買いたい物を選ぶ。
4. **🛒買い物** … 献立表＋日用品から買い物リストを自動生成。材料を合算し、家族の人数で分量を自動換算。各品を「在庫あり」で除外、店（ツルハ/ビッグ/その他）を割り当て、「店ごとに買い物」モードでチェックしながら買う。

## ✅ 変えていいところ（デザインのみ）

- `<style>` の中身（色・フォント・余白・角丸・影・レイアウト・アニメーション等）
- 各ボタンや見出しの見た目、アイコン絵文字、配色テーマ
- 画面の雰囲気（もっと可愛く/シンプル/モダン に、など自由に）
- CSSクラスを **増やす** のは自由

## ⛔ 絶対に変えてはいけないところ（触ると壊れます）

1. **`<script>` の中のJavaScriptロジックは変更しない**（関数の中身・計算・データ処理はそのまま）
2. **`const SEED = [...]`（約190品の献立データ）と `const GOODS_SEED`（日用品）は1文字も変えない**
3. **`const KEY="matomegai1"`（データ保存先の名前）は変えない**　※変えると今までの保存データが消えます
4. **HTMLの id は変えない**：`view` `nav-week` `nav-dish` `nav-goods` `nav-shop` `overlay` `modal` `flash` `imp`
5. **JavaScriptが使っているclass名は消さない・改名しない**（下記リスト）。装飾を足すのはOKだが、これらの名前は残すこと：
   `btn` `sub` `gray` `danger` `small` `wide` `card` `note` `search` `catbar`
   `day` `sat` `sun` `dname` `chips` `chip` `x` `addbtn`
   `dishrow` `nm` `ings` `arrow` `goodsgrid` `gbtn` `sel` `del`
   `item` `stocked` `done` `info` `inm` `iq` `storechip` `stockbtn` `on` `checkbox` `storehead`
   `mhead` `ingrow` `n` `q` `u` `rm` `pickrow` `added` `flash` `show` `row2`
6. **`onclick="..."` などの属性はそのまま残す**（ボタンの動作がここに書かれています）

## 守ってほしい技術的な決まり（重要）

- **外部ファイル・CDN・ライブラリを一切使わない。** インターネットに繋がってなくても、ファイルをダブルクリックしただけで動く「1枚完結」を必ず守る。
  - ❌ `<link href="https://...">`、`<script src="https://...">`、Webフォント読み込み、外部画像URL は禁止
  - CSSもJSも全部この1ファイルの中に書く（今と同じ形）
- **スマホ縦画面ファースト**（横スクロールが出ないこと）。指でタップしやすいボタンサイズを保つ。
- **日本語UI**のまま。
- ライト/ダークどちらか一方でOK（今はライト）。凝るならダーク対応も歓迎だが必須ではない。
- 完成したら **`index.html` 全文をそのまま**（省略なし）で返してください。「以下同様」などで省略しないこと。
